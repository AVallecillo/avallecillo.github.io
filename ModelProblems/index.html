Model Problems in Model-Driven Web Engineering

Introduction

It is common for a discipline, especially during its establishment and consolidation phases, to adopt some shared, well-defined problems for teaching, study, and for analysing different solution and strategies to common problems. Often known as model systems or type problems, they provide a way to compare methods and results, work out new techniques on standard examples, and set a minimum standard of capability for new participants. Model problems also provide a pre-debugged source of educational exercises. This initiative is not new, and has successfully been used in some disciplines -- in particular in software architecture.

The Web Engineering community would also benefit from sharing set of standard example problems. These would improve our ability to work out ideas, exhibit techniques, and compare results. Some of these problems have already been used in several workshops and events (e.g., IWWOST'01, MDWE2005) to stimulate discussions focusing on concrete examples. Each of these is somehow part of the common language of discourse in the field, and each example provides a familiar concrete instance that illustrates an important set of issues.

These pages are intended to put together a set of model problems that range from very simplistic ones (that deal with very particular concepts and mechanisms of Web Engineering) to full-fledged applications. Each one tries to show different aspects of typical Web systems, focusing on specific concerns (navigation, integration with external systems, etc.). Furthermore, the applications proposed in the examples can be seen as representative systems of different kinds of Web applications, such as interactive systems (Products Catalogue), data-intensive systems (PetStore), or workflow-based applications (Travel Agency and the Conference Review System).

Current model problems

  • Simple (static) web site
            A web application that allows navigation between static pages (like this one).
  • A simple Address Book
            A web application that allows content publication, in this case an Address Book with the details of a set of contacts.
  • An Address Book with searches
            A web application that allows the extraction of object content from the domain objects and the selection of the attributes to publish, based on a parameter generated by the user’s interaction. .
  • An Address Book with content updates
            A web application that allows the the creation, extraction, deletion and modification of object content from the domain objects.
  • Products Catalogue
            A web application to display the product catalogue and content management system of a shop.
  • A stock market monitor
            A transient web application that displays the quotes of selected stocks of an exchange market, using external Web Services.
  • Research publications
            A web application that displays several portlets with the publications of a given author.
  • PetStore
            A web-based application that provides customers with online shopping.
  • The Travel Agency System
            A web application used by a travel agency for selling travels to its customers.
  • Conference Review System
            A web system to support the process of submission, evaluation and selection of papers for a conference.
