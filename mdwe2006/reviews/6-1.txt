The paper presents an overview of the Magrathea case study, which is performed in the context of a running project at the authors univerisity. Albeit a high level description, the paper presents an interesting case-study of process-based web application modeling and generation, and if combined with a demonstration on the workshop, I think it could be an asset for the workshop.

- Summary and main contributions:

The paper present an overview of the Magrathea case-study, performed within the PESOA project. The framework offers a set of tools which enables developers to describe domain specific artifact (at implementation side), implementing (business) processes. Furthermore, the framework offers the possibility to define process-models which, given the concrete artifacts, allow generate the concrete web application.

   
- Weak points of the paper:

* elaboration of related work is poor

* explanation remains quite high-level



- Strong points of the paper:

* the paper overviews a complete approach at hand of a case study

- Any suggestions for improvements: 

* it would be interesting, also in the context of the workshop, to compare this process-driven web application generation approach, with existing (model-driven) web engineering approaches (e.g. OOHDM, WebML, WSDM, Hera, etc.)

* similarly, comparison with existing process or task-based (generation) approaches may be added

* in the printed version, figures are hardly readable. Please make them larger.
